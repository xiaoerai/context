# 历史住客信息缓存

> 状态：🔧 开发中

---

## 概述

用户每次入住都要重新上传身份证、填写姓名和证件号，体验差。在 User 记录上关联历史住客信息，返回用户时自动填充，支持切换不同住客。

---

## 数据结构

**User 表** 新增字段：
```typescript
{
  // ...现有字段
  guestIds: string[]  // 关联的住客ID列表（按最近使用排序）
}
```

**Guest 表**：不变，已有按 idNumber 去重逻辑。

---

## 交互流程

```
进入 checkin 页面
      ↓
  调 API 获取历史住客
      ↓
  ┌─────────────────────────────────┐
  │  历史住客（点击切换）            │
  │  [张三 ✓]  [李四]  [+ 新住客]   │
  ├─────────────────────────────────┤
  │  上传身份证                      │
  │  姓名：张三（已自动填充）         │
  │  证件号：110***（已自动填充）     │
  │                                 │
  │       [提交]                    │
  └─────────────────────────────────┘
```

- 点击历史住客 → 切换表单内容
- 点击「+ 新住客」→ 清空表单
- 上传新身份证 → OCR 覆盖表单内容
- 提交时如果是新住客，后端自动创建并关联到 User

---

## API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/user/guests` | GET | 获取当前用户的历史住客列表 |

**响应**：
```json
{
  "success": true,
  "data": [
    { "name": "张三", "idNumber": "110101199001011234" },
    { "name": "李四", "idNumber": "110101199002022345" }
  ]
}
```
